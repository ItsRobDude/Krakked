## 2024-05-23 - [The Giant Method Antipattern] **Observation:** The `execute_plan` method in `oms.py` was acting as a "God Method", mixing high-level orchestration with low-level business logic (exposure calculation). This made the core execution flow hard to follow and the logic hard to reuse. **Improvement:** Extracted `_calculate_projected_exposure` into a dedicated private method. This reduces the cognitive load of `execute_plan` significantly, transforming it from a wall of math into a clear sequence of steps: "Check Age -> Filter Actions -> Calc Exposure -> Loop".

## 2025-12-20 - [The Giant Method Antipattern] **Observation:** The `run_cycle` method in `StrategyEngine` (src/kraken_bot/strategy/engine.py) was a "Giant Method", mixing high-level orchestration (data checks, persistence, risk routing) with a dense, 60-line double-loop for gathering and summarizing intents from strategies. This made the high-level flow of the decision cycle hard to see. **Improvement:** Extracted the intent collection loop into `_collect_intents`. This separates the "what" (orchestrating the cycle) from the "how" (iterating strategies and handling exceptions), making the main method readable at a glance.

## 2025-05-21 - [The Giant Method Antipattern] **Observation:** The `parse_app_config` function in `src/kraken_bot/config_loader.py` had grown to over 370 lines, validating every configuration section inline. This made it difficult to locate specific validation logic and increased the risk of unintended side effects when modifying one section. **Improvement:** Extracted the UI configuration parsing logic (approx. 80 lines) into a dedicated private helper `_parse_ui_config`, and the integer validation logic into `_validate_config_int`. This modularization improves readability and allows for easier testing of specific configuration sections.

## 2026-01-16 - [Repetitive API Boilerplate] **Observation:** The `fetchJson` and `fetchJsonStrict` functions in `ui/src/services/api.ts` repeatedly implemented the same Authorization header logic. This duplication added noise and made the specific error-handling logic of each function harder to spot. **Improvement:** Extracted a `getHeaders` helper. This cleanly separates the "how" of authentication from the "what" of the request lifecycle, making the API layer more declarative and easier to maintain.
