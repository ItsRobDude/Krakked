## 2025-05-15 - [Refactoring Giant Request Methods] **Observation:** The `_request` method in `KrakenRESTClient` mixes low-level HTTP handling, authentication logic, and extensive error parsing in a single 80-line block. This makes it hard to scan for the "happy path" or understand error handling in isolation. **Improvement:** Extract error handling into `_handle_api_error` and use `session.request` to unify HTTP method dispatch, reducing the main method to a clear linear flow.
