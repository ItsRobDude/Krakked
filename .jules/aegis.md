## 2025-04-03 - [Timing Attack in AuthMiddleware Token Validation]
**Vulnerability:** The API token verification in `AuthMiddleware` used a simple string equality check (`!=`) to compare the provided Authorization header with the expected token. This allowed attackers to potentially guess the token character by character by measuring the response time.
**Learning:** In Python, standard string comparisons (`==`, `!=`) return immediately upon finding the first mismatched character, creating timing side-channels for security-sensitive checks like token validation.
**Prevention:** Always use `secrets.compare_digest` for security-sensitive string comparisons. For robustness, particularly with headers which might contain unexpected characters, encode both strings to bytes (e.g., `utf-8`) before comparison, and ensure neither input is `None` (by defaulting to empty strings). If there is a local `secrets.py` file, import the standard library module with an alias, like `import secrets as std_secrets`.
