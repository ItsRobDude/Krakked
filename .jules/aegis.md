## 2024-03-30 - Fix timing attack vulnerability in auth token validation
**Vulnerability:** A simple string equality check (`==` / `!=`) was used for validating UI authentication tokens, which leaks the length of the matching prefix through timing differences.
**Learning:** The Python standard library `secrets` module, which contains `compare_digest` for constant-time comparisons, is shadowed in this repository by `src/kraken_bot/secrets.py`. To use the standard library `secrets` module, it must be aliased on import (e.g., `import secrets as std_secrets`). In addition, `secrets.compare_digest` requires string/bytes inputs, and encoding string header values to bytes avoids `TypeError`s with non-ASCII characters.
**Prevention:** When validating security tokens or secrets, use `compare_digest` from the standard library `secrets` module (using an alias like `import secrets as std_secrets`) and ensure inputs are utf-8 encoded strings/bytes, explicitly handling `None` values (e.g. `or ""`).
